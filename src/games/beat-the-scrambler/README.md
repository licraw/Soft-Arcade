# Beat the Scrambler

Beat the Scrambler is a neon sliding-number puzzle hosted inside the Soft Arcade portal. Players choose a difficulty, solve the board with directional moves, and submit a short arcade-style name to the leaderboard when they win.

This game started as a standalone jQuery app and is now mounted from a React wrapper so it can live inside the Next.js game portal.

## Gameplay

The board is a classic sliding puzzle. One tile space is empty, and each move slides a neighboring tile into that gap. The goal is to restore the grid to numeric order.

Difficulty levels:

- `Easy`: 3x3 board
- `Medium`: 4x4 board
- `Hard`: 5x5 board

Controls:

- Desktop: arrow keys
- Mobile: swipe in the direction you want to move

The game tracks:

- current level
- move count
- elapsed time
- local personal best per difficulty

When a run is completed, the player can submit a short name to the shared leaderboard.

## Interface

The UI has two modes:

- Desktop: full HUD with stats and action buttons visible in the top bar
- Mobile: compact header with a collapsible game menu

The top bar includes:

- leaderboard
- restart level
- return to main menu

Restarting or leaving an active run requires confirmation to avoid accidental progress loss.

## Leaderboard Behavior

The leaderboard is anonymous and arcade-style. There are no user accounts or sign-in flows.

Each score submission includes:

- player name
- difficulty
- move count
- completion time

Scores are ranked by:

1. fewest moves
2. fastest time

The Worker applies basic validation and a small per-IP cooldown to reduce spam. It is not intended to be cheat-proof.

## Module Layout

Files:

- `BeatTheScramblerGame.tsx`: React wrapper that installs jQuery and mounts the legacy game
- `mountBeatTheScrambler.js`: game lifecycle, DOM event binding, board rendering, score submission, and modal behavior
- `game-config.js`: API configuration and level constants
- `game-utils.js`: puzzle helpers
- `jquery-3.4.1.js`: vendored jQuery runtime used by the legacy implementation
- `styles.module.css`: scoped game styles, including global selectors for the mounted DOM
- `index.ts`: public module export for the game registry

The old standalone files were folded into this module. `game.html` at the repository root is a legacy reference and is not the portal route.

## Portal Integration

The game is registered in `src/games/registry.ts` with:

- `id`: `beat-the-scrambler`
- `slug`: `beat-the-scrambler`
- `type`: `vanilla`
- component: `BeatTheScramblerGame`

The live route is generated by the shared game page at:

```text
/games/beat-the-scrambler
```

## Cloudflare Backend

Beat the Scrambler owns the current Cloudflare Worker and D1 leaderboard implementation. The schema is specific to this game's score model: difficulty, move count, and completion time.

- `GET /api/scores?level=easy|medium|hard&limit=10`
- `POST /api/scores`

Example score payload:

```json
{
  "name": "AAA",
  "level": "medium",
  "moves": 82,
  "time": 64
}
```

The D1 schema and Worker live outside the game folder:

- `worker/src/index.js`
- `worker/migrations/0001_create_scores.sql`
- `wrangler.toml`

The implementation is a good starting point for future game leaderboards, including Near Miss, but those games should get their own score schema and API contract instead of sharing Beat the Scrambler's level/move/time assumptions directly.

## Standalone Cloudflare Deployment

Beat the Scrambler also has legacy standalone deployment support through Cloudflare Pages and the Worker. That path is separate from the main Soft Arcade portal deployment.

Relevant scripts:

```bash
npm run dev:worker
npm run db:migrate:local
npm run db:migrate:remote
npm run deploy:pages
npm run deploy:worker
```

If the standalone frontend is not served from the same origin as the Worker, update the API URL in `game-config.js`.
